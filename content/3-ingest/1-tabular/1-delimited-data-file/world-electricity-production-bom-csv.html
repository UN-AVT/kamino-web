Ingest a csv file with BOM encoding

Photo by Michael Dziedzic on Unsplash

Every decoding is another encoding…
— David Lodge

The first character is a magical character, invisible to the human eye, but readible by a computer. It is the byte order mark (or BOM) and it’s telling the computer that the characters that follow are encoded in Unicode. This issue will often arise when saving to a CSV file from a spreadsheet.

Ingest

without considering BOM

df_no_bom <- read.csv("archetypes/world-electricity-production/world-electricity-production-bom.csv", header = TRUE, stringsAsFactors = FALSE)
df_no_bom
names(df_no_bom)
## [1] "ï..RowIDs"    "Year"         "Measure"      "Country_Name" "Country_Code"
## [6] "Series_Name"  "Series_Code"

Ingest

recognizing BOM encoding

df_bom <- read.csv("archetypes/world-electricity-production/world-electricity-production-bom.csv", header = TRUE, stringsAsFactors = FALSE, fileEncoding = 'UTF-8-BOM')
df_bom
names(df_bom)
## [1] "RowIDs"       "Year"         "Measure"      "Country_Name" "Country_Code"
## [6] "Series_Name"  "Series_Code"
