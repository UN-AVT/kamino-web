Validate

Verify the conformity and quality of source Data Set

Data Validation is the process of ensuring that the data conforms to the requirements of the use cases. Typical examples include ensuring that the column First Name only contain alphabets, column Age contains only whole numbers, and the column Date is set up in a Date Format (MM/DD/YY). The goal of Data Validation is to ensure that data is consistent, accurate and complete.

These are the key areas we’ll focus on addressing:

  • Data-Type Check : verify that the characters in the dataset conform to the expected characters or datatypes
  • Range and Constraint Validation : check the consistency with a maximum/minimum range or evaluate a sequence of characters
  • Code Check : verify that data conforms to an external rule or organizational rule (eg. Country Codes of a Region)

Data validation determines the formal aspects and rules of the data, however does not necessarily ensure the accuracy of the data. Errors such as misspellings and inaccurate information can still occur. Therefore, data validation is an important aspect of data science before other processes are operated upon the data.

These are some sample functions that we’ll be utilizing:

  • Is Alpha : check that the cells contains letter and letters only
  • Is Integer : check if the cells are whole numbers or with the integer datatype
  • Country Code : Validate Country Names and Country Codes using global standards
